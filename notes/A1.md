## Concept in my own words

A socket is an interface for process-to-process communication using file descriptors. Sockets communicate with each other via a connection channel. The Kernel binds a port

- A file descriptor is an integer for accessing an I/O resource (files, sockets, pipes)
- A Socket Stream is used for TCP connections
- A Socket Datagram is used for UDP connections
- Encapsulation: how each layer in the layered network model adds its headers to the input, before it goes doen to the next layer
- Big Endian: Storing the most significant byte before the least significant byte, so that that it reads from left to right. Used in Network Transfer
- Little Endian: Storing the least significant byte first. Used by some Intel chips
- addrinfo: metadata about the socket address; pointer to socket address
- sock_addr: address family (e.g. AF_INET) and sa_data (inline bytes)
- sock_addrin: sock_address copy for IPv4
- IPv4 -> 4 bytes, 32 bits; IPV6 -> 16 bytes, 128 bits
- communication is through `send()` and `recv()` calls

## Build summary

- It's easy to create a socket, knowing the address family and the socket stream type. Python has enums for both and a rich socket module

**Q1:** What is the difference between a socket and a connection?
**A:** A socket is an endpoint. A connection is an established channel of communication between two sockets.

**Q2:** What does the OS do when you call `bind()`?
**A:** check if `(protocol, address, port)` already exists. If it doesn't, it creates a mapping between the socket and the `pap`. Then it associates the socket with that local address so future `recv()` calls are sent to the socket's address.

**Q3:** What happens if the port is already in use, and why?
**A:** If the port is already in use, an OSError "Address already in use" error message is returned because a port can only be bound to one socket. The kernel maintains a unique mapping of `(protocol, address, port)` so they can't bind to the same port
